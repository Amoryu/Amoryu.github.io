# src和href的区别

  1. 请求资源类型不同

    • href是Hypertext Reference的缩写，表示超文本引用。用来建立当前元素和文档之间的链接。常用的有：link、a。

    • 在请求 src 资源时会将其指向的资源下载并应用到文档中，常用的有script，img 、iframe；

  2. 作用结果不同

    • href 用于在当前文档和引用资源之间确立联系；

    • src 用于替换当前内容；

  3. 浏览器解析方式不同

    • 若在文档中添加href ，浏览器会识别该文档为 CSS 文件，就会并行下载资源并且不会停止对当前文档的处理。这也是为什么建议使用 link 方式加载 CSS，而不是使用 @import 方式。 s
    • 当浏览器解析到src ，会暂停其他资源的下载和处理，直到将该资源加载、编译、执行完毕，图片和框架等也如此，类似于将所指向资源应用到当前内容。这也是为什么建议把 js 脚本放在底部而不是头部的原因。

# link标签和@import标签的区别

  • link是XHTML标签，除了加载CSS外，还可以定义RSS等其他事务；@import属于CSS范畴，只能加载CSS。

  • link引用CSS时，在页面载入时同时加载；@import需要页面网页完全载入以后加载。

  • link是XHTML标签，无兼容问题；@import是在CSS2.1提出的，低版本的浏览器不支持。

  • link支持使用Javascript控制DOM去改变样式；而@import不支持。

# 媒体标签

  • 图片标签

    < img >标签，单标签，需要添加标签属性（属性名=“属性值”）

    <img src=" " alt=" ">

    属性

    • src：图片路径
      • 绝对路径：从盘符开始的目录下的绝对位置
      • 相对路径：从当前文件开始出发到目标文件的位置
        • 上级目录：../pic.jpg ( ../返回上一级，返回几级就用几个../,比如../../pic.jpg)
        • 同级目录：./pic.jpg或pic.jpg
        • 下级目录：image/pic.jpg
    • alt：替换文本，加载不出图片时的替换文本
    • title：提示文本，鼠标悬停在图片上时的文本显示
    • width和height：只设置其中一个，另一个会等比例缩放（图片不会变形）

  • 音频标签

    < audio></ audio>音频标签

    <audio src=""></audio>

    属性

    属性功能
    src音频路径
    controls显示播放的控件
    autoplay自动播放（部分浏览器不支持）
    loop循环播放

    目前支持三种音频格式：.mp3、.wav、.ogg

  • 视频标签

    < video>< /video>视频标签

    <video src=""></video>

    属性

    属性功能
    src视频的路径
    controls显示播放的控件
    autoplay自动播放（Chrome需配合muted实现静音才有效果）
    loop循环播放

    目前支持三种视频格式：.mp4、.webM、.ogg

# 链接标签

< a>< /a>链接标签：也叫a链接，超链接，锚链接，行内元素

<a href="./目标网页.html"></a>

属性

属性作用
href跳转地址（#表示空链接，不跳转）
targetblank：在新窗口跳转（保留原网页）
self：默认值，在当前窗口中跳转（覆盖原网页）

# 列表标签

# 1、无序列表

< ul>< li>< /li>< /ul>：每个列表项前默认用圆点标识

< ul>：只允许包含< li>标签

< li>：可以包含任意内容

<ul>无序列表的整体
    <li>列表项1</li>
    <li>列表项2</li>
</ul>

# 2、有序列表

< ol>< li>< /li>< /ol>：每个列表项前默认用序号标识

< ol>：只允许包含< li>标签

< li>：可以包含任意内容

<ol>
    <li></li>
    <li></li>
</ol>

# 3、自定义列表

< dl>< dt>< /dt>< dd></ dd>< /dl>：

<dl>	自定义列表的整体
    <dt>自定义列表列表主题</dt>
    <dd>列表内容项</dd>		<!--相当于<li>-->
</dl>

< dl>：只允许包含< dt>< dd>标签

< dt>< dd>：可以包含任意内容

# 合并单元格

跨行合并（水平合并），跨列合并（垂直合并）

遵循左上原则：保留左上单元格，删除其他

==技巧==：

  • 跨行合并rowspan：找要合并的行的< tr>中对应位置的< td>
  • 跨列合并colspan：删除当前< td>的下一个或多个< td>
属性属性值作用
rowspan合并的单元格个数多列的单元格合并
colspan合并单元格的个数多行的单元格合并

注意：只能在砼结构标签中才能合并，不能跨解构标签合并
