# 选择器

（CSS属性）加上大括号{ }

  1. # 标签选择器：

    将CSS属性应用到所有这类标签

    <div></div>
//CSS样式
div{
	....
}

  2. # 类选择器：class，设置页面中所有带这个类名的标签

    类名可以由数字，字母，中划线，下划线组成，但不能以数字和中划线开头

    一个标签可以由多个类名，以空格隔开

    <div class="yangshi"></div>
//CSS样式
.class{
	...
}

  3. # ID选择器：id，设置页面中带这个id属性值的标签

    在页面中，id是唯一的，不可重复
    一个标签只能有一个id，一个id只能选中一个标签

    <div id="yangshi"></div>
//CSS属性
#yangshi{
	...
}

  4. # 通配符选择器：*号，设置页面中所有标签

    *{
    ...
}

  5. # 属性选择器

    选中包含指定属性[attr]的标签E，适用于一些HTML布尔属性

    E[attr]{
	...
}

    选中属性和属性值完全匹配[attr=''value'']的标签E， 例如单选框 或者 复选框：

    E[type="radio"]{

}

    （可以在不想被覆盖的CSS属性后面加上important）

# 权重叠加计算

判断某个样式在多个复合选择器内的哪个选择器生效，需要进行权重比较

（行内的样式个数，id选择器的样式个数，类选择器的样式个数，标签选择器的样式个数）

计算出每个复合选择器的上述个数 例如（0，1，1，0）

  1. 然后对每个复合选择器的个数结果 逐级比较大小，若比较出来就停止
  2. 如果数字都相同，则比较层叠性，谁在下面谁优先级大
  3. 如果加了! important**（不是继承）**，则优先级最高
  4. 继承的优先级最低
  5. 都是继承进行比较，哪个继承的父级离自己近（先继承谁），哪个优先级就高

# 外边距折叠现象

  • # 合并现象

    给两个盒子都设置外边距margin，相邻的部分会合并，变为设置外边距的两倍

  • # 塌陷现象

    设置子元素的外边距margin，会设置到父元素上（子元素想下移，自己没，把他爹拽下去了）

    ==原因，父子标签，且都是块级元素，子元素的外边距会影响父元素==

    解决方法

    1. 父元素设置border或padding

    2. # 父元素设置overflow：hidden;

    3. 子元素转成行内块元素

# 行内元素的内外边距问题

  • 行内元素的外边距margin只对左右生效

  • padding对内容上下不生效，但是会撑开元素，超出的部分不会

  • 设置行内元素的行高line-height，实现padding对上下边距的效果

# 盒子实际大小

盒子大小 = 内容content的大小 + 边框border的大小 + 内边距padding的大小

CSS2之前，盒子实际大小需要手动减

CSS3开始，支持盒子大小自动内减，设置好盒子的宽高大小后，不论后期是加padding还是border，都会自动内减为设置的宽高大小（前提是加的内边距padding和边框border不能超过盒子大小）

box-sizing：border-box

==（放在* { css } 里面，实现页面所有盒子大小自动内减少）==

# 浮动

相对于父元素浮动

早期的作用：图文环绕（给图片添加float属性）

现在的作用：网页布局（让块级元素在行内排列）

# 特点

  • 会脱离标准流，在标准流中不占位置

  • 浮动比标准流高半个级别，可以覆盖标准流中的元素，（会覆盖盒子，不覆盖文字）

  • 浮动找浮动：下一个浮动的元素会自动找上一个浮动的元素（顶部对齐）

  • 一行显示多个，可以设置宽高（具备行内块元素的特点）

  • 不能再通过margin：0 auto；居中

    ==浮动没给高度会导致脱标==

# 清除浮动

清除浮动带来的影响

如果子元素浮动了，此时子元素不能撑开标准流的块级元素

原因：子元素浮动后脱标（脱离标准流），不占位置

  1. # 额外标签法

.clearfix:after{
    clear:both;

# 相对定位relative

自恋型定位：相对于自己之前的位置进行移动，之前的位置仍然保留（没有脱标）

  • 需要配合方位属性实现移动 left:100px;

    应用场景

  • 配合绝对定位使用**（子绝父相）**

  • 用于小范围的移动

    ==同时出现相对的方位，以左left和上top为准==

# 绝对定位absolute

拼爹型定位：相对于非静态定位的父元素进行定位移动

寻找父级元素：就近原则，逐级往上找，直到浏览器可视区域

  • 如果有父级但没定位，或者没有父级，则以浏览器可视区域进行移动（实际开发一般不以浏览器可视区域进行定位）

  • 需要配合方位属性实现移动

  • 脱标，不占位置，具备了行内块的特点，加宽高会生效，若没加宽高也没有有内容，宽高为0（保证盒子有宽和高，或者有内容）

  • 不能使用margin：0 auto进行水平居中

    应用场景

  • 配合相对定位使用（子绝父相）

# ==绝对定位的元素实现 水平居中==

.box{
    position:absolute;
    left:50%;							//水平居中
    margin-left:-(盒子的宽度的二分之一);
    
    top:50%;
    margin-top:-(盒子高度的二分之一);	//垂直居中
}

  • # 盒子尺寸需求变更时：配合属性==transform：translate(-50%,-50%);== 将位移设置为自身宽度高度的一半

# 固定定位fixed

死心眼型定位：永远相对于浏览器可视区域进行定位移动（让盒子固定再屏幕中的某个位置）

  • 需配合方位属性实现移动
  • 在页面中不占位置，脱标
  • 具备行内块特点（保证盒子有宽和高，或者有内容）

# CSS精灵图

项目中将多张小图片，合并成一张大图片，称之为精灵图

优点：减少服务器发送次数，减轻服务器的压力，提高页面加载速度

# 精灵图使用步骤

  1. 创建一个盒子，设置盒子尺寸和小图片尺寸相同（盒子一般都用行内元素）
  2. 将精灵图设置为盒子的背景图片

# 改变背景图的位置

background-postion: 水平 垂直；

# 背景图的大小

设置背景图的大小

属性名：background-size： 宽度 高度

取值场景
数字+px常用
%百分比相对于自身宽高的百分比
contain将图片等比例缩放直到填满盒子（宽或者高碰到边缘则停止缩放，导致图片显示不全）
cover刚好填满整个盒子，没有空白

# 文件和目录准备

  • # 新建项目文件夹xxx-pc-client，所有相关文件放在这

    1. 项目文件夹不建议使用中文
    2. 复制favicon.ico到这
    3. 复制images目录和uploads目录到这
      1. images：存放网站固定使用的图片素材，如logo，样式修饰图片....
      2. uploads ：存放非固定使用的图片素材，如商品图片，宣传图片...
    4. 新建index.html放到这
    5. 新建 .css文件，建议新建以下CSS文件（引入时也按此顺序）
      • base.css ：基础公共样式，（ul的样式，margin和padding等）
      • common.css ：多个相同模块的重复样式（网页头部，底部）
      • index.css ：首页的样式
